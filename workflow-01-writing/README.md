# 工作流 ① · AI 写作全流程

> 从「灵光一闪」到「可发布文章」的完整链条

---

## 流程图

```
灵感碎片 → 选题判断 → 结构搭建 → AI 扩写 → 人工深加工 → 品牌排版 → 发布
  (2min)    (5min)     (10min)    (15min)    (30-60min)    (5min)     (2min)
```

---

## 步骤 1：捕获灵感（2 分钟）

**工具**：手机便签 / 微信"文件传输助手" / flomo

**原则**：不判断，不编辑，只记录。灵感是易腐品。

```
格式：[一句话灵感] + [为什么你觉得有意思]

例：
"AI 越强，人越焦虑——为什么？"  →  因为朋友圈都在转 AI 取代人类的文章
"孤独不是病"                    →  昨晚失眠想到的
```

---

## 步骤 2：选题判断（5 分钟）

把灵感丢给 AI，让它帮你判断是否值得写：

```markdown
## Prompt 模板 · 选题判断

我有一个写作灵感：「____」。

请帮我从以下维度分析这个选题：

1. **读者覆盖面**（窄/中/宽）
   - 谁会关心这个话题？大概多少人？
   
2. **独特角度**（列出 3 个）
   - 别人通常怎么写这个话题？你能写出什么不一样的角度？
   - 每个角度一句话描述

3. **内容类型适配**（教程/观点/故事/分析/清单）
   - 这个选题最适合哪种形式？为什么？

4. **竞争程度**（低/中/高）
   - 搜索这个话题，已经有多少好内容了？

5. **融合潜力**
   - 能不能和热点事件、经典思想、或你的专业领域结合？

最后给出：✅ 值得写 / ⚠️ 调角度后值得写 / ❌ 暂时搁置
```

---

## 步骤 3：结构搭建（10 分钟）

确定选题后，用 SCQA 框架搭骨架：

```markdown
## Prompt 模板 · 结构搭建

我要写一篇关于「____」的文章，选择的角度是「____」。
目标读者是「____」。

请用 SCQA 框架帮我搭建结构：

S (Situation 情境)  —— 读者现在的状态/认知
C (Complication 冲突) —— 有什么问题/矛盾/意外
Q (Question 问题)   —— 核心问题是什么
A (Answer 答案)     —— 我的回答（分 2-3 层展开）

要求：
- 标注每部分预估字数（总长控制 1200-1800 字）
- 标注金句插入位置（至少 3 处，用 ✦ 标记）
- 标注路标位置（每 300 字一个视觉断点）
- 开头必须有一个 3 秒钩子
- 结尾必须回扣开头

输出格式：
## 钩子（0-50字）
## S 情境（~200字）
## C 冲突（~200字）
## Q 问题（~100字）
## A 答案（~800字）
  ### 第一层（~300字）← 金句位置 1
  ### 第二层（~300字）← 金句位置 2
  ### 第三层（~200字）← 金句位置 3
## 收尾回扣（~150字）
```

---

## 步骤 4：AI 辅助扩写（15 分钟）

有了结构骨架，逐段让 AI 扩写：

```markdown
## Prompt 模板 · 段落扩写

基于以下结构要点，扩写成完整的段落：

**段落主题**：「____」
**预估字数**：~300 字
**在文章中的位置**：第 X 层展开
**需要包含**：
- 一个具体案例或场景（不是纯说理）
- 与核心命题的逻辑关联
- 一段金句（用 ✦ 开头，可独立传播）

**写作风格要求**：
- 口语化但不口水化
- 每段不超过 4 行（手机屏幕）
- 抽象概念配具象比喻
- 不用"首先其次最后"，用自然过渡

**参考金句风格**：
「✦ 你的价值从来不在你的技能里——在你的眼光里」
```

## 步骤 5：人工深加工（30-60 分钟）⚠️ 最关键

> 🚨 AI 写的初稿，必须经过人工深加工。这是区分「AI 写的文章」和「人用 AI 写的文章」的关键步骤。

### 深加工清单

- [ ] **去 AI 味**：删掉"在当今时代""随着...的发展""总而言之"等 AI 高频套话
- [ ] **加个人经验**：至少插入 1 段自己的真实经历/观察
- [ ] **概念一致性检查**：确认核心术语全文统一（如"认知能量"不能混用"心力"）
- [ ] **节奏调整**：长短句交替，AI 倾向写等长句子
- [ ] **钩子打磨**：开头 50 字重写 3 遍，选最好的
- [ ] **金句打磨**：每句金句至少改 3 遍，去掉所有"也许""大概""可能"
- [ ] **朗读检查**：大声读一遍，不顺口的地方就是需要改的
- [ ] **删除冗余**：AI 喜欢把同一个意思说三遍——删掉两遍

详见 **[工作流 ② · 品牌排版](../workflow-02-branding/README.md)**

## 步骤 7：发布前自检（2 分钟）

- [ ] **标题** ≤ 9 中文字
- [ ] **摘要** ≤ 17 中文字
- [ ] **核心命题句**：遮住全文，只看 thesis 那一句——能独立站住吗？
- [ ] **开头钩子**：3 秒能让人停下来吗？
- [ ] **路标清晰**：每 300 字有视觉断点（小标题/金句）
- [ ] **金句可传播**：至少有 1 句读者愿意截图转发
- [ ] **闭环完成**：结尾回扣开头了吗？
- [ ] **排版完整**：6 区块全部到位
- [ ] **中文编码**：API 推送时 `ensure_ascii=False` + `charset=utf-8`

## 真实案例对比

### ❌ 改之前 · 《生育焦虑的背后》（草稿）

> 当社会在为出生率下降而焦虑时，这场对话早已超越了人口统计学。讨论的起点看似简单，却层层递进。第一层是人口结构、养老负担。第二层是社会自救、技术替代。第三层是人活着究竟为了什么。生育焦虑从来不只是关于生不生的问题。这场对话没有标准答案。但能够这样思考，本身就证明了人不仅仅是繁衍的载体。

**问题**：无钩子、无章节分区、无金句、无个人经验、术语不统一、"这场对话"重复 3 次。

### ✅ 改之后 · 《生育焦虑的第三层》

> （钩子）你焦虑的不是生不生——是生了之后，你是谁。
>
> （壹 · 第一层）人口数字在说话。养老金缺口、劳动力萎缩、消费降级——这些是你看得到的焦虑。但它们只是表层。
>
> （贰 · 第二层）社会在自救。鼓励生育的政策一波接一波，技术乐观主义者说 AI 会补上劳动力的缺口。但这一层仍然在问"怎么让人多生"——而不是"为什么人不愿意生"。
>
> （叁 · 第三层）我32岁那年，坐在阳台上盯着对面的楼看了两个小时。不是在想生不生的问题。是在想：如果我所有的身份——员工、儿子、朋友——都被拿走，我还剩什么？后来我意识到，那是我第一次真正面对自己。
>
> ✦ 真正的生育率问题，不在子宫里——在意义里。当一个人连自己的存在意义都找不到的时候，你让她怎么敢把另一个生命带到这个世界？
>
> （收尾）下次讨论生育率的时候，不妨换个问题：不是"怎么让人多生"，而是"我们给了年轻人什么值得活的人生"。

**改进点**：钩子前置、章节分区、金句可传播、个人经验插入、术语统一。

## 常见问题

**Q：AI 扩写后还需要人工改多久？**
A：初稿出来，人工至少花 30 分钟深加工。低于这个时间，文章会有 AI 味。

**Q：什么时候不适合用 AI 辅助写作？**
A：当你自己对选题没有真正思考过的时候。AI 可以帮你表达，不能帮你思考。

**Q：如何避免文章"AI 味"太重？**
A：三个信号——① 套话多（"在当今""总而言之"）② 句子等长 ③ 没有具体场景。删套话、调节奏、加故事。
